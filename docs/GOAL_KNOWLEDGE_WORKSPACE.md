# Goal — 知识库写作台（对标语雀 / 飞书文档 / FlowUs）

> **调用（持续执行）**  
> `/goal 按 docs/GOAL_KNOWLEDGE_WORKSPACE.md 执行。从阶段 A 开始，严格 Issue → branch → PR → CI → merge，一阶段一 PR；每阶段验收 DoD 后再进入下一阶段。遇到 Vercel/GitHub 真机问题当场修。直到阶段 F 完成。`

> **产品一句话**  
> 把「能发文章的博客后台」升级成「能像文件夹一样堆文档、能轻松改旧文、写得爽的知识库写作台」——前台仍是博客/实验室，**后台心智对齐语雀 / 飞书文档 / FlowUs**，而不是再堆 WordPress 插件。

## 0. 用户痛点（本 Goal 必须解决）

| # | 痛点（用户原话） | 根因（当前实现） | 目标体验 |
|---|------------------|------------------|----------|
| P1 | 能写，**不好改以前的文章**，好麻烦 | 后台只是扁平列表；改文要进 `/admin` → 找标题 → `/edit`；Vercel 上旧文列表可能滞后；无最近编辑、无树导航、无一键续写 | 左侧知识树 / 最近编辑 / 搜索，一点即开；Ctrl+S 反复保存；打开即最新内容 |
| P2 | **不能像文件夹一样堆放** | 内容模型是 `content/posts/*.md` 平铺；只有 tags / series，没有目录树 | 支持 **知识库目录（文件夹）**：嵌套、拖拽排序、一篇可在树中定位 |
| P3 | 整体「一点都不好」相对语雀/飞书/FlowUs | 缺：侧栏树、页内大纲、块感编辑、回收站、快捷键、模板、双链弱 | 分阶段补齐 **作者台** 能力；读者站保持简洁 |

### 明确不做（防范围爆炸）

| 不做 | 原因 |
|------|------|
| 多人大文档协同实时光标（CRDT/OT） | 单作者 + Git 审计已够；协同属另一产品 |
| 完整 IM / 多维表格 / 看板 / 表单 | 飞书套件过重 |
| 付费空间 / 组织多租户 / 权限矩阵 | 个人站 |
| 抛弃 Markdown / 改存私有二进制 | 必须继续 Git 可 diff |
| 重做公共站点为 Notion 公开站 | 前台仍是博客 + Lab（HF/RSS） |

## 2. 内容模型升级（核心）

### 2.1 现状

```text
content/posts/
  foo.md
  bar.md          # 平铺，无目录
```

### 2.2 目标模型（Git 友好）

```text
content/
  tree.json                 # 知识树：文件夹 + 文档顺序（可生成）
  posts/
    intro.md
    notes/
      week-1.md
      deep-dives/
        transformer.md
```

**规则**

| 规则 | 说明 |
|------|------|
| 物理路径 | 文档可落在 `content/posts/**/*.md` 子目录 = 文件夹 |
| slug | 仍全局唯一；默认由路径末段生成；URL 仍 `/blog/{slug}`（稳定，不因搬家改链） |
| tree.json | 描述侧栏展示顺序、折叠、重命名文件夹显示名；**可与真实目录双向同步** |
| frontmatter | 保留 title/date/tags/draft/pinned/series/cover；新增可选 `folder` 显示覆盖 |
| 搬家 | 后台拖进文件夹 = `git mv` 等价（GitHub API：读旧 + 写新 + 删旧） |

**兼容**

- 旧 `content/posts/*.md` 全部视为「根目录」文档，零迁移成本  
- 读路径：递归 `**/*.md`；写路径：按当前树节点目录  

### 2.3 与 series / tags 关系

| 概念 | 用途 |
|------|------|
| **文件夹** | 作者组织（像电脑目录） |
| **tags** | 读者检索、交叉主题 |
| **series** | 连载顺序（已有可保留） |

三者并存，不互斥。

## 3. 信息架构（后台 / 前台）

### 3.1 后台：写作台（新 `/admin` 壳）

```text
┌────────────┬──────────────────────────────┬─────────────┐
│ 知识树     │  编辑区                        │ 属性 / 大纲 │
│ · 搜索     │  大标题                         │ 发布状态    │
│ · 最近     │  Markdown 宽屏编辑器            │ slug/日期   │
│ · 树       │  底栏：保存 | 取消 | 状态提示   │ tags/系列   │
│ · 回收站   │                                │ 封面        │
└────────────┴──────────────────────────────┴─────────────┘
```

**路由建议**

| 路由 | 用途 |
|------|------|
| `/admin` | 写作台首页：最近 + 树概览 |
| `/admin/workspace` | 完整三栏工作台（可与上合并） |
| `/admin/posts/new?folder=` | 在指定文件夹新建 |
| `/admin/posts/[slug]/edit` | 编辑（保留深链；工作台内嵌同组件） |
| `/admin/trash` | 回收站（软删） |

### 3.2 前台（读者）

| 能力 | 目标 |
|------|------|
| `/blog` | 可按文件夹筛选；默认最新流 |
| `/docs` 或 `/kb` | **可选** 知识库树浏览（像语雀公开库）— 阶段 D |
| 文章页 | 面包屑：库 / 文件夹 / 标题；上一篇下一篇按树序 |
| Lab | HF / RSS 保持，不塞进文件夹模型 |

## 4. 功能清单（分优先级）

### 阶段 A — 改旧文不再痛苦（Must，先做）

> 不解决「文件夹」也能先让「改文」顺手。

| ID | 功能 | 验收 |
|----|------|------|
| A1 | **后台列表读 GitHub 最新**（与编辑页一致），避免改完刷新变旧 | 改标题保存后，列表立刻见新标题（或明确「同步中」） |
| A2 | **最近编辑**（按 `updatedAt` frontmatter 或 GitHub commit 时间） | 打开 `/admin` 顶部可见最近 10 篇，一点进入编辑 |
| A3 | **列表强化**：搜索标题/slug/正文摘要、筛选草稿/发布、排序（更新/创建） | 20 篇内 2 秒找到目标 |
| A4 | **编辑器顶栏**：大标题可点改 + 复制前台链接 + 打开前台 | 无需回列表 |
| A5 | **保存态**：底栏常驻成功/失败；Ctrl+S 多次覆盖；导入 md 后保存再刷新内容正确 | 回归现有修复 + 列表一致 |
| A6 | **未保存离开提示**（beforeunload / 路由拦截） | 有脏内容时浏览器提示 |

**DoD A**：用户从后台 1 次点击进入任意旧文并保存，无需记忆 URL；刷新后内容与 GitHub 一致。

---

### 阶段 B — 文件夹 / 知识树（Must，核心）

| ID | 功能 | 验收 |
|----|------|------|
| B1 | 内容模型支持 `content/posts/**/*.md` | 子目录文章可读写、可渲染 |
| B2 | `content/tree.json` 或等价索引 + 同步脚本 | 树与磁盘一致 |
| B3 | Admin **侧栏树**：展开/折叠、选中高亮 | 像语雀左侧 |
| B4 | **新建文件夹**、**重命名文件夹**、**删除空文件夹** | GitHub 提交可见 |
| B5 | **文档移入/移出文件夹**（菜单或拖拽 v1 可用菜单） | slug 不变，URL 不变 |
| B6 | 在文件夹下 **新建文档** | 默认路径正确 |
| B7 | 树排序（同级 up/down 或 drag） | 刷新保序 |
| B8 | 前台文章面包屑 | 文件夹路径可见 |

**DoD B**：至少 2 级文件夹；文档可在文件夹间移动；旧平铺文自动在根目录。

---

### 阶段 C — 写作台一体化（Should）

| ID | 功能 | 验收 |
|----|------|------|
| C1 | 三栏布局（树 | 编辑 | 属性）宽屏可用；窄屏可折叠 | 1080p 不挤 |
| C2 | 文档内 **大纲**（H2/H3 跳转） | 长文可跳 |
| C3 | **⌘/Ctrl+K** 命令面板：搜文档、新建、去前台 | 键盘流 |
| C4 | **模板**：空白 / 周记 / 读书笔记 / 论文笔记 | 新建可选 |
| C5 | 导入 .md 可指定目标文件夹 | 导入+落盘路径对 |
| C6 | 导出当前 .md 下载 | 本地备份 |

**DoD C**：不打开浏览器其它页也能完成「找文 → 改 → 存 → 再找」。

### 阶段 D — 前台知识库体验（Should）

| ID | 功能 | 验收 |
|----|------|------|
| D1 | `/kb` 或 `/docs` 树形浏览（仅已发布） | 访客可点树阅读 |
| D2 | `/blog` 增加「按目录」视图切换 | 流 / 目录两种 |

### 阶段 E — 安全删除与版本（Should）

| ID | 功能 | 验收 |
|----|------|------|
| E1 | **软删除** → `content/trash/` 或 `draft: true` + `deletedAt` | 误删可恢复 |

| E3 | 「在 GitHub 查看历史」外链 | 跳 blob/commits |
| E4 | 可选：展示最近 5 次 commit message（API） | 只读 |

**DoD E**：删文不再心惊；恢复路径清晰。

### 阶段 F — 打磨与质量（Must 收尾）

| ID | 功能 | 验收 |
|----|------|------|
| F1 | 空状态文案（无文档 / 无文件夹） | 有引导 |
| F2 | 移动端 Admin 可用（树抽屉） | 手机能改文 |
| F3 | 权限：未登录不可见树与 API | 已有 session 延伸 |
| F4 | 自动化测试：树 CRUD、slug 稳定、GitHub mock 或 smoke | CI 绿 |
| F5 | 更新 README + 本 Goal 勾选 | 文档与代码一致 |
| F6 | 清理：旧入口跳转、死链 | 无 404 |

**DoD F**：`npm test` + `npm run build` + 主路径手测通过；可宣布「知识库写作台 v1」。

## 5. 技术约束与策略

| 约束 | 策略 |
|------|------|
| Vercel 只读盘 | 所有写 → GitHub Contents API（已有）；**读编辑数据优先 GitHub**（已部分做，列表/树全面铺开） |
| Git 适合文本 | 树用 json；文档用 md；大图仍 `/uploads` |
| 部署延迟 | 保存成功提示区分「GitHub 已提交」vs「前台 CDN 约 1 分钟」；Admin **不依赖**新部署读旧文 |
| 包体 | 编辑器继续 MD 方案（@uiw 或后续可换）；不引入完整 Notion 引擎 |
| 交付节奏 | **一阶段一 PR**；Issue 标题 `feat(kb): 阶段X …` |

### 建议目录 / 模块

```text
src/lib/content-tree.ts      # 树读写、路径解析
src/lib/posts.ts             # 递归扫描、slug 索引
src/app/admin/(workspace)/   # 新写作台布局
src/components/admin/        # Tree, CommandPalette, Outline
content/tree.json
content/posts/**/*.md
```

## 6. 里程碑与顺序（执行顺序锁死）

```text
A 改文体验  →  B 文件夹树  →  C 写作台壳  →  D 前台知识库  →  E 回收站  →  F 打磨
```

- **不可先做 D 再做 B**（前台树依赖模型）  
- **C 可与 B 后半重叠**，但 B1–B3 必须先合入  
- 每阶段结束：merge 后打 tag 可选 `kb-phase-A` …

## 7. 验收总表（Goal 完成定义）

全部满足即 Goal 完成：

1. **改旧文**：从 Admin 最近列表 / 搜索 / 树任一入口 ≤2 次点击打开；保存后刷新内容正确（GitHub 为准）。  
2. **文件夹**：≥2 级目录；文档可移动；slug/前台 URL 稳定。  
3. **堆放感**：侧栏树 + 前台面包屑或 `/kb` 树，用户感知「像文件夹」。  
4. **多次保存**：Ctrl+S 无「slug 已存在」误报；成功提示在底栏且可理解。  
5. **导入 md**：可导入到当前文件夹；保存后刷新不丢。  
6. **质量**：CI 绿；硬约束（Markdown、Git、Vercel）不破。  
7. **非目标**未偷偷做进协同/多租户。

## 8. 每阶段标准作业（强制）

1. `gh issue create` 写清阶段 ID 与验收勾选  
2. `git checkout -b feat/kb-phase-X`  
3. 实现 + `npm test` + `npm run lint` + `npm run build`  
4. PR 描述含：用户痛点 / 方案 / 测试计划  
5. CI 通过后 merge；删远程分支  
6. 在本文件阶段表将对应 ID 勾为 `✅`（小 PR 可只改 docs）  
7. 进入下一阶段（无需再问「是否继续」，直到 F 或 blocker）

### Blocker 才暂停

- 无 `GITHUB_TOKEN` 写权限  
- GitHub API 限流导致无法测  
- 用户明确改优先级

## 10. 成功画像（给未来的自己）

打开站点后台，**左侧是文件夹树**，最近改过的笔记在最上面；点进一篇，**右边立刻是上次的 Markdown**，改一句 Ctrl+S，提示「已保存」；再开一个文件夹堆论文笔记；前台访客可以按目录浏览，也可以继续刷 HF / RSS。
